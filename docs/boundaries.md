# Boundaries

`lupine.science` is a public doorway, not the science system.

## This Repo Owns

- the homepage copy and links
- public metadata files
- static assets required by the homepage
- nginx serving config
- Cloud Run deploy workflow
- a static verifier for link and claim hygiene

## This Repo Links To

- `library.lupine.site` for reports, evidence notes, status labels, and papers
- `lupi.live` for molecular viewing and shared structures
- the science/control-plane repo for source, proofs, experiments, and ledgers
- the public `glim-think` progress API for compact live-loop status

## This Repo Must Not Own

- claim adjudication
- article bodies
- formal proof source
- generated experiment evidence
- MLIP policy or promotion rules
- viewer code
- investor or gated material

## Copy Rules

Keep copy durable and modest:

- say what surface to open next
- keep public paper status factual and conservative
- prefer links to evidence over summary claims
- avoid temporary launch copy that will age badly
- avoid metrics unless they are fetched live or pinned to a cited source

## Forbidden Public Claims

Do not state or imply:

- paper accepted, submitted, or peer-reviewed
- journal or venue assignment
- scientific proof where the Library/repo does not show a proof status
- production readiness for research loops that have only local proof

The static verifier catches common wording regressions, but review the page by
reading it too. The embarrassing bugs usually arrive as plausible sentences.
